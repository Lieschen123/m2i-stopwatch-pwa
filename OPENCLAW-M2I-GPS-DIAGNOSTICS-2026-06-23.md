# M2I GPS Diagnostics Patch - 2026-06-23

## Summary

Finished the GPS diagnostics patch so the UI no longer presents a local GPS estimate as meaningful until at least one GPS sample has been accepted.

The privacy model is unchanged: GPS remains opt-in per workout, route points stay in memory only, route/coordinate fields are not stored or shared, and public Nostr sharing remains a separate redacted event.

## Changed Files

- `src/gps.js`
  - Tracks secure context, geolocation availability, watch state, accepted/rejected sample counts, and last GPS error.
  - Exposes `status()` for live diagnostics without finalizing a workout summary.
  - Keeps route points memory-only and clears the last point on stop.

- `src/main.js`
  - Shows live GPS diagnostics during a workout.
  - Shows `Waiting for first accepted GPS sample...` or `No accepted GPS samples yet.` instead of a false `0.000 km local estimate`.
  - Shows a warning on the signed claim screen when GPS was attempted but produced no accepted samples.

- `src/claim.js`
  - Keeps no-sample GPS attempts as diagnostics in the private signed claim.
  - Only includes `distance_meters`, `distance_km`, `local_verification`, and `verification_method` when at least one GPS sample was accepted.
  - Keeps public projection redaction behavior unchanged.

- `src/styles.css`
  - Adds styling for workout GPS diagnostics and no-sample claim warnings.

- `tests/core.test.mjs`
  - Adds coverage for no-sample GPS claims, public projection behavior, tracker startup diagnostics, and poor-sample rejection.

- `scripts/build.mjs`
  - Changes generated deployment headers from `geolocation=()` to `geolocation=(self)` so opt-in GPS can work on the deployed same-origin PWA.

- `dist/_headers`, `dist/assets/main.js`, `dist/assets/main.css`
  - Regenerated by `npm run build`.

## Verification

- `npm test` passed: 15 tests passed.
- `npm run build` passed.
- Built bundle: `dist/assets/main.js` 100879 bytes, 35965 gzip bytes.
- Generated `dist/_headers` now contains `Permissions-Policy: geolocation=(self), microphone=(), camera=(), payment=()`.

## Remaining Deployment Requirement

GPS still requires a secure browser context. Deploy the PWA over HTTPS, or test on a browser exception such as `localhost`. A plain HTTP deployment will continue to report `this page is not in a secure context` and will not start GPS.
